lifeline runs the full dev lifecycle ‚ÄĒ spec ‚Üí plan ‚Üí build ‚Üí review ‚Üí test ‚Üí merge ‚ÄĒ as
gated phases that produce real artifacts a human can read: a spec, a plan, reviews, an

runs the same on Claude Code and Cursor. ūü™Ę

## ūüß† How it works

```mermaid
flowchart LR
    S([spec]) --> P([plan]) --> B([build<br/>waves]) --> R([review]) --> T([test]) --> M([merge])
    M --> RD[/reviewer_doc.md/]
    M --> QD[/qa_doc.md/]
```

A feature moves through six gated phases. At each one, lifeline dispatches a **focused
role** ‚ÄĒ spec-writer, architect, implementer, reviewer, test-runner ‚ÄĒ that role returns a
structured payload, and the orchestrator writes a **durable artifact** to disk before
advancing. You approve (or override) at every gate. Nothing happens off the record: every
gate, retry, verdict, and override lands in an append-only `flow.md`.

| Phase | What happens | Leaves behind |
|---|---|---|
| ūüďĚ **Spec** | Intent ‚Üí testable spec. Open questions surfaced, never assumed. | `spec.md` |
| ūüóļÔłŹ **Plan** | 2‚Äď3 approaches proposed, one chosen, work split into dependency-ordered waves. | `plan.md` |
| ūüĒ® **Build** | TDD per task (RED‚ÜíGREEN‚ÜíREFACTOR). Independent tasks run in parallel on full tier. | `task.md` |
| ūüĒé **Review** | One review per task, four lenses ‚ÄĒ logic, architecture, security, performance ‚ÄĒ depth scaled to effort. | `review.md` |
| ūüß™ **Test** | Tests run; coverage measured on *changed* files; every gap ‚Üí a smoke-checklist line. | `test_result.md` |
| ūüöÄ **Merge** | Invariant check, override audit, smoke gate ‚Üí two handoff docs, then branch action. | `reviewer_doc.md`, `qa_doc.md` |

**Two ways in, same engine:**

- **ūüí¨ Ambient** ‚ÄĒ skills auto-trigger on intent. *"review this"* fires a four-lens review,
  *"are we sure this works?"* fires coverage‚Üísmoke. Each skill stands alone ‚ÄĒ no command
  required.
- **ūüé¨ Orchestrated** ‚ÄĒ one command runs the whole gated cycle with persisted state and a
  short setup wizard. Close your laptop mid-cycle; it cold-resumes from disk on any harness.

Both paths execute the *same* skills ‚ÄĒ no duplicated logic, no drift. ūüß¨

## ‚ú® What makes it different

1. **ūüďä Coverage‚Üísmoke seeding.** Passing tests aren't proof ‚ÄĒ 152 can pass while the real
   handler is mocked and never runs. lifeline measures coverage on the *changed* files and
   turns every gap into a line in a human smoke checklist at merge. Advisory, never silent.
2. **ūü§Ě Reviewer + QA handoff docs.** Merge emits `reviewer_doc.md` (code-visible: changes
   mapped to requirements, risks, override audit) and `qa_doc.md` (blackbox: feature flow,
   numbered test cases, smoke checklist). The humans actually know what changed.
3. **ūüĒć An auditable trail.** Every gate, retry, verdict, and override lands in an
   append-only flow log. Overrides are legal ‚ÄĒ but they resurface at the merge gate.
   No skeletons stay in the closet. ūüíÄ

## ūüß≠ Philosophy

1. **Roles return payloads; the orchestrator writes artifacts.** Every dispatch ends in a
   structured payload, persisted to disk *before* the cycle advances. Nothing is implied.
2. **Degrade, never block.** Every capability has a documented fallback. A degraded tier
   keeps 100% of the methodology and artifacts ‚ÄĒ it only loses hard-gate *enforcement*
   (‚Üí self-checks) and parallelism (‚Üí sequential). That's the whole tradeoff.
3. **Coverage-honest.** Passing tests are necessary, not sufficient ‚ÄĒ line coverage can't
   see over-mocking. Gaps surface (advisory, never blocking) and route to a human smoke
   gate. ūüēĶÔłŹ
4. **Auditable.** `flow.md` records every gate, retry, verdict, and override. Overrides
   are legal ‚ÄĒ but they're never silent, and they resurface at merge. ūüíÄ
5. **Human handoff is part of the lifecycle.** A cycle isn't done at green tests. It's done
   when a reviewer and a QA engineer can each pick up a purpose-built document. ūü§Ě
